## Skills System

Skills are reusable knowledge modules that AI agents can invoke to get detailed instructions for specific tasks. They live in the `skills/` directory.

### Meta-Skills

These skills help you manage the skills system itself:

| Skill | Description | URL |
|-------|-------------|-----|
| `skill-creator` | Create new AI agent skills | [SKILL.md](skills/skill-creator/SKILL.md) |
| `skill-sync` | Sync skills to AGENTS.md tables | [SKILL.md](skills/skill-sync/SKILL.md) |

### Auto-invoke Meta-Skills

When performing these actions, ALWAYS invoke the corresponding skill FIRST:

| Action | Skill |
|--------|-------|
| Creating new skills | `skill-creator` |
| After creating/modifying a skill | `skill-sync` |
| Regenerate AGENTS.md skill tables | `skill-sync` |
| Troubleshoot missing skills in AGENTS.md | `skill-sync` |

## Commit & Pull Request Guidelines

Follow conventional-commit style: `<type>[scope]: <description>`

**Types:** `feat`, `fix`, `docs`, `chore`, `perf`, `refactor`, `style`, `test`

Before creating a PR:
1. Complete checklist in `.github/pull_request_template.md`
2. Run all relevant tests and linters
3. Link screenshots for UI changes
